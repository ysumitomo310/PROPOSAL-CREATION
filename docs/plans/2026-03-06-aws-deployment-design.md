# AWS デプロイ設計書

> 作成日: 2026-03-06
> ステータス: 承認済み

## 背景・目的

ProposalCreation をローカル環境から AWS EC2 へ移行し、会社の全メンバーが利用できる状態にする。

## 決定事項

| 項目 | 決定内容 | 理由 |
|------|---------|------|
| インスタンスタイプ | t3.xlarge（4vCPU / 16GB） | Neo4j + LLM処理でメモリ余裕が必要 |
| OS | Amazon Linux 2023 | AWS推奨。長期サポート |
| リージョン | ap-northeast-1（東京） | 国内ユーザー向け低レイテンシ |
| アクセス制限 | セキュリティグループで会社グローバルIPのみ許可 | VPN代替。社内ネットワーク隔離 |
| URL | EC2 Elastic IP（IPアドレス直打ち） | ドメイン取得不要。情シス依頼不要 |
| プロトコル | HTTP（ポート80） | 社内ネットワーク限定のため許容 |
| シークレット管理 | .envファイルをEC2上に配置（SSHアクセス者のみ参照可） | シンプル。管理者以外アクセス不可 |
| バックアップ | なし（初期フェーズ） | 後日追加を検討 |
| コード取得 | GitHub公開リポジトリからgit clone | rsync不要。デプロイが再現可能 |
| 稼働時間 | 平日9:00〜19:00のみ自動起動・停止 | 月額約$55に削減（常時稼働比70%削減） |

## アーキテクチャ

```
インターネット
    |
    | (会社グローバルIPのみ許可)
    |
[AWS Security Group]
    |
[EC2 t3.xlarge]
├── Nginx (port 80)
│   ├── / → Frontend (port 3001)
│   └── /api → Backend (port 8000)
├── Docker Compose
│   ├── Frontend (Next.js :3001)
│   ├── Backend  (FastAPI :8000)
│   ├── PostgreSQL (:5432 内部のみ)
│   └── Neo4j (:7687 内部のみ / :7474 管理者IPのみ)
└── EBS (データ永続化)
```

## セキュリティグループ設定

| ポート | プロトコル | ソース | 用途 |
|------|-----------|-------|------|
| 22 | TCP | 管理者IP | SSH |
| 80 | TCP | 会社グローバルIP | HTTP（アプリ） |
| 7474 | TCP | 管理者IP | Neo4j Browser（任意） |

※ PostgreSQL(5432)・Neo4j Bolt(7687)・Backend(8000)・Frontend(3001) は外部に非公開

## 月額コスト見積もり

| 項目 | 金額 |
|------|------|
| EC2 t3.xlarge（220時間/月） | 約 $45 |
| EBS gp3 100GB | 約 $8 |
| Elastic IP（稼働時間中は無料） | $0〜$4 |
| **合計** | **約 $53〜57/月** |

## 不採用アプローチと理由

| アプローチ | 不採用理由 |
|-----------|----------|
| Approach A（バックアップなし t3.large） | メモリ8GBはNeo4j+LLMで不足リスク |
| Approach C（Secrets Manager + CloudWatch） | 1〜5人規模にはセットアップコストが高い |
| 独自ドメイン + HTTPS | 情シスへのDNS変更依頼が必要。初期フェーズでは不要 |
