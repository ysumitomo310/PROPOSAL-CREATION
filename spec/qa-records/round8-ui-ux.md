# Round 8: UI/UX設計 - Q&A記録

> **ステータス**: 完了
> **日付**: 2026-02-17

## Q1. 画面構成・ナビゲーション

- **決定**: ハイブリッド（ダッシュボード + ステップインジケーター）
- **理由**: 案件ダッシュボードで全案件の進捗を俯瞰しつつ、各案件詳細画面にGate 1→2→3のステップインジケーターを配置。自由な画面遷移と現在地の明示を両立。年間20-30案件の並行管理に対応。

## Q2. Gate 2（マッピング結果レビュー）画面

- **決定**: テーブル + サイドパネル（マスター/ディテール）
- **内容**:
  - 左: フィルタ・ソート付きテーブル（要件名 / 判定 / 確信度 / 重要度 / ステータス）
  - 右サイドパネル: 選択行の詳細（検索根拠、proposal_text、関連ノード、LangSmithトレースリンク）
  - フィルタプリセット: 「要レビュー（Must×Low）」「承認済み」「全件」
- **理由**: 数百件規模にはテーブルUIが最も実用的。一覧性を保ちつつ画面遷移なしで詳細確認・修正が可能。

## Q3. リアルタイム処理フィードバック

- **決定**: ストリーミング結果表示（SSE）
- **内容**:
  - マッピング完了した要件から順次テーブルに表示（Server-Sent Events）
  - 全体進捗バー（`85/200件`）も併設
  - ユーザーは処理中でも完了分のレビューを開始可能
- **理由**: FastAPIのStreamingResponse + SSEで実装可能。LangGraphの各ステップ完了をイベントとして発火。待ち時間が実質ゼロ。

## Q4. Gate 3（提案構成レビュー）画面

- **決定**: アウトライン（左）+ プレビュー（右）の2ペイン
- **内容**:
  - 左ペイン: セクションツリー（ドラッグ&ドロップで並替可能）+ 各セクションへのコメント入力
  - 右ペイン: 選択セクションのproposal_text内容プレビュー + Markdownプレビュー
  - セクション単位で「承認」「再生成リクエスト（コメント付き）」のアクション
- **理由**: Gate 2（テーブル+サイドパネル）と同じ操作パターンでUI一貫性を維持。

## Q5. デザインシステム・UIライブラリ

- **決定**: shadcn/ui + Tailwind CSS
- **追加ライブラリ**:
  - TanStack Table: テーブル（フィルタ・ソート・仮想化）
  - dnd-kit: ドラッグ&ドロップ（セクション並替）
- **理由**: Next.js + shadcn/uiは生成AI開発で最も生成精度が高い組合せ。ベースコンポーネントはshadcn/uiで統一し、高機能部品のみ専用ライブラリを追加。
