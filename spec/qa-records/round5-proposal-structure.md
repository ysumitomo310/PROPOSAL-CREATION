# Round 5: 提案骨子生成の設計 - Q&A記録

> **ステータス**: 完了
> **日付**: 2026-02-17

## Q1. 過去提案書の構造化方式

- **決定**: 2層構造（構成テンプレート + セクション内容）
- **理由**: 構成パターン自体は大きく変わらないが、業種・規模ごとの微妙な違い（セクション順序、厚みの配分）を保持するために2層で管理。

## Q2. 提案構成の生成ロジック

- **決定**: LLM生成 + テンプレート参照
- **理由**: RFPの評価基準やマッピング結果の強みに応じて柔軟に構成を調整。過去テンプレートはRAGで参考提供。

## Q3. マッピング結果→提案書への変換

- **決定**: 業務領域集約 + 強み訴求セクション
- **補足**: 提案書の定番セクション「貴社システム要求に対するご回答」「要求事項に対するソリューションの実現方法」にproposal_textを織り込む形式。業務領域ごとにグルーピングし、標準対応率が高い領域を「本提案の強み」として訴求。

## Q4. 提案書のトーン・スタイル制御

- **決定**: スタイルガイド + Few-shot
- **理由**: 自社の提案スタイルに合わせたガイドと過去の良い表現例で品質を安定化。Gate 3で人間がトーン含めレビュー。

## Q5. 提案書の反復改善フロー

- **決定**: セクション単位の差分編集（基本）、将来的に対話型リファインメントに拡張
- **理由**: フィードバック時に該当セクションのみ再生成。他の承認済みセクションは維持。
