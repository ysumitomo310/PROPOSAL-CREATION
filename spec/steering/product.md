# Product Overview - ERP提案書作成支援システム (ProposalCreation)

## プロダクト概要

ERP導入を提案・推進するプライムSIerとして、顧客から受領するRFP（提案依頼書）の情報をINPUTに、RAG技術を活用して提案書・見積書を効率的に作成するAIアシスト型アプリケーション。

## 対象ERP製品

- **GRANDIT** - 国産ERP
- **SAP S/4 HANA Public Edition** - グローバルERP

## ビジネスコンテキスト

- 我々はERPの提案・導入をプライムで行う会社である
- 顧客からRFPを受領し、それに基づいて提案書・見積書を作成する
- **年間RFP対応件数**: 20〜30件
- 現状、提案書作成は属人的で時間がかかるプロセスである
- AIとRAGを活用し、提案品質の標準化と作成速度の向上を目指す

## 主要ステークホルダー

- **提案チーム**: 提案書を作成するメンバー（主要ユーザー）
- **営業**: RFPを受領し提案プロセスを開始する
- **経営層**: 提案品質と受注率の向上を期待

## 解決すべき課題

1. RFP分析の属人化 - ベテランの知見に依存
2. 提案書作成の長時間化 - 毎回ゼロベースに近い作業
3. 機能要件マッピングの手間 - ERP標準機能との照合作業
4. 過去提案のナレッジ散逸 - 蓄積はあるが検索・活用が困難

## 現状の運用と移行背景

現在はNotebookLM + Gemini GEMで以下を実施:
- RFP資料をNotebookLMにアップロード
- GEMとのやりとりで機能マッピング一覧を作成

**課題**: Excel直接取込不可、チャンク戦略制御不可、出力の構造化制御に限界、案件ごとの再構築が必要

## Phase 1 PoC スコープ

| 項目 | 内容 |
|------|------|
| **対象** | マッピングエンジンのみ（RFP分析・提案生成はPhase 2以降） |
| **対象ERP** | SAP S/4 HANA Public Edition（1製品に絞って検証） |
| **ナレッジ** | SAP全モジュールのScope Item + モジュール紹介資料をフル投入 |
| **INPUT** | 過去の実RFP 1案件分（機能要件一覧Excel） |
| **正解データ** | 同案件で実際に作成した機能マッピング結果 |

### Phase 1 成功基準

| 指標 | 基準 |
|------|------|
| **マッピング精度** | 人間の判定との一致率 > 70%（5段階判定レベル） |
| **処理速度** | 1要件あたり平均 < 30秒 |
| **コスト** | 200要件の処理で < $50（LLM API費用） |
| **提案文品質** | proposal_textが提案書に転記可能なレベル（定性評価） |
| **トレーサビリティ** | 全結果にScope Item ID + LangSmithトレースリンク |
